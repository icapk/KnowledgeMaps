# 亿级科技知识图谱系统 — 产品概览

---

## 一、产品定位（一句话）

> 面向科研管理与产业决策的**亿级多模态科技知识图谱一体化服务平台**，整合论文、专利、学者、企业数据，依托图谱核心算法实现人才关联挖掘、成果溯源、产业链协同研判。

---

## 二、核心价值（一句话）

> 打通"科学家"与"产业链"之间的数据壁垒，让科研管理者和产业分析师能够**一站式完成人才精准画像、隐性合作关系发现、产业链全景态势感知及重大事件预警研判**，将原本碎片化、低效率的人工情报分析工作提升到分钟级自动化水平。

---

## 三、本产品与市面类似产品的核心区别

| 对比维度 | AMiner（清华） | 企查查/天眼查 | Semantic Scholar | **本系统** |
| :--- | :--- | :--- | :--- | :--- |
| **数据底座** | 偏学术：论文+学者 | 偏商业：工商+投融资 | 偏文献：论文+引证 | **学术+工商+产业链多模态融合** |
| **关系挖掘深度** | 1-2度合作网 | 股权穿透 | 引证关联 | **支持最大6度多跳路径寻优 + 双点贡献度量化** |
| **同事/校友推理** | 弱（仅机构标签） | 强（高管任职） | 无 | **基于时间轴+机构匹配算法的精准推理构建** |
| **产业链全景图** | 无 | 强（但无学术底层） | 无 | **上中下游结构化建模 + 学术/专利/企业节点融入** |
| **事件监测预警** | 科技资讯 | 企业风险监控 | 无 | **TOP-N高影响力事件筛选 + 产业链波及传导分析** |
| **核心壁垒算法** | 学者消歧 | 股权穿透 | NLP语义关联 | **消歧 + PageRank + 路径寻优 + 贡献度模型 全栈覆盖** |
| **专业驻场服务** | 无 | 无 | 无 | **10人团队驻场2年，全流程图谱运维** |

**一句话差异总结：**
> 市面产品"偏学术"或"偏商业"单线作战，本系统**独创性地将学术人才网与产业链商业网在同一张知识图谱上融合**，是目前唯一同时满足"人才评估 + 产业研判 + 事件预警"三位一体需求的系统。

### 本产品相较市面通用知识图谱产品的三大核心优势

**优势一：一个平台看全部，不用到处找**
> 通用知识图谱产品往往只做单一数据源的实体-关系抽取（如纯文献图谱或纯工商图谱），数据之间缺乏跨域关联。本系统在底层将**学术论文、专利权族、工商股权、产业链结构、政策事件**五大异构数据统一建模到同一张图谱中，实现了"从科学家到产业链"的全链路贯通，用户无需在多个平台间反复切换比对。

**优势二：系统自动发现关系，不靠人猜**
> 通用图谱产品大多停留在"搜索 + 可视化"层面，本系统内置了**6度多跳路径寻优、PageRank核心度排名、任职时间轴精准匹配推理、双点贡献度量化评估**等一系列专业图算法引擎，能够自动发现用户凭肉眼和经验无法察觉的隐性关联（如跨机构的潜在同事关系、间接人脉传导链路），将知识图谱从"被动查询工具"升级为"主动发现引擎"。

**优势三：查完直接导出用，不用再整理**
> 通用图谱产品输出通常是一张网络图或一份实体列表，用户仍需自行分析研判。本系统的每个模块均以**"结构化清单 + 多条件查询 + 一键导出"**作为标准输出规范，所有分析结果可直接导出为研究报告或决策依据，打通了从"数据洞察"到"行动决策"的最后一公里，真正服务于科研管理和产业规划的实际工作流。

## 四、典型用户画像及需求场景

### 用户1：科研管理人员

| 维度 | 描述 |
| :--- | :--- |
| **角色定义** | 高校科研处、国家级科研机构的科研管理负责人 |
| **核心痛点** | 评估团队学术实力时，只能看到论文/专利表面数据，无法可视化专家完整的科研协作关联网络和全球学术影响力排名 |
| **用户需求** | 快速看清一位专家的真实学术地位和关系网 |
| **产品使用场景** | 输入学者名字，一键生成全景关系图谱 |
| **期望价值** | 传统人工调研需要2-3天完成的专家背景报告，系统可在分钟级内自动生成结构化画像 |

---

### 用户2：产业/投资分析师

| 维度 | 描述 |
| :--- | :--- |
| **角色定义** | 产业基金、科技投资机构的行业研究员及风控人员 |
| **核心痛点** | 难以透视一家初创科技企业背后真正的技术护城河：不知道它与哪些顶尖高校有深度合作、核心专利发明人是否是业界权威、竞品在产业链中的位置 |
| **用户需求** | 看清一家企业背后真实的科技实力 |
| **产品使用场景** | 输入企业名，穿透生成底层科创支撑网 |
| **期望价值** | 过去需要靠产业链报告+问询+多个平台交叉查验的工作流，现在在一张图谱上完成闭环 |

---

### 用户3：政府科技主管部门决策者

| 维度 | 描述 |
| :--- | :--- |
| **角色定义** | 地方科技局、发改委、工信局等政府部门的科技创新与产业规划决策人员 |
| **核心痛点** | 制定区域科技发展规划和招才引智政策时，缺乏对本地产业链技术短板、外部顶尖人才储备及竞争区域态势的全局量化视角，决策依据碎片化 |
| **用户需求** | 找到本地产业链短板和能补上的人 |
| **产品使用场景** | 调取产业链全景图，定位薄弱环节精准招引 |
| **期望价值** | 将过去依赖专家座谈和人工调研的产业规划方式，升级为数据驱动的精准决策模式，显著提升政策制定的科学性和时效性 |
